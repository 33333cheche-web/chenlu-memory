# 智研康养平台 AI 智能体群组（Agentic Workflow）架构与核心设计方案

**版本**：V4.0 (商业交付·合规融合版)
**模块**：前端 AI 智能体应用
**定位**：基于 AI 与知识库（RAG）技术的全链路康养服务闭环（筛查 -> 评估 -> 建议 -> 转化）

> ⚠️ **合规性声明（核心红线）**：
> 本系统定位为**“日常康养与体质评估工具”**，并非医疗器械。所有前端智能体的话术、界面及报告，均严禁出现“诊断”、“问诊”、“处方”、“主治医生”等医疗专属词汇，规避一切医疗责任风险。

---

## 〇、 总体架构设计与商业闭环模型

本项目采用 **Agentic Workflow（智能体工作流）** 架构。系统由 4 个专精不同领域的智能体节点组成，接力为用户提供连贯的服务。

底层依赖统一的**用户上下文记忆模块（Context Memory）**。用户在各个智能体之间的对话记录和体质数据可无缝流转。在 C 端游客模式下依然能实现数据留存，并最终在核心价值节点引导注册，完成高转化率的私域沉淀。

---

## 一、 核心智能体（Agent）深度功能设计

### 1. 交互式体质测试智能体（Agent 1：The Interviewer）
**【模块定位】**：多模态体质信息采集前台（像一位耐心细致的健康顾问）。
**【业务目标】**：降低传统长表单带来的用户流失率，通过拟人化的对话完成关键体征指标的提取。

*   **多模态交互框架（文字+组件）**：
    *   在对话流中动态抛出 UI 卡片（如程度选择滑块、症状多选按钮），实现“文本+卡片”的混合交互，降低用户输入成本。
*   **环境参数注入（自带温度的感知）**：
    *   系统实时获取当前节气与天气注入提示词（Prompt）。Agent 交互时会自然带出：“今日立秋且天气干燥，您最近是否觉得口干或者皮肤紧绷？”
*   **非线性追问引擎**：
    *   摒弃死板顺序，利用大模型的推理能力进行动态追问。例如用户提到“经常失眠”，Agent 懂得顺着追问“是入睡困难还是容易早醒？”，直至预设的体质判定关键信息收集完毕。

### 2. 体质报告生成智能体（Agent 2：The Analyst）
**【模块定位】**：基于 RAG（检索增强生成）技术的智能分析中枢。
**【业务目标】**：将零散的测试对话，结合康养知识库，映射为结构化、高颜值的“体质评估报告”（非医疗诊断书）。

*   **RAG 知识图谱融合**：
    *   接收 Agent 1 传递的数据，通过向量检索（Vector Search）匹配后台的中医养生/现代营养学知识库，推导出用户的核心体质倾向（如痰湿、气虚）。
*   **前端结构化可视化**：
    *   Agent 严格输出 JSON 格式数据，由前端渲染为美观的数据看板（健康雷达图、体质标签环）。
*   **情绪共鸣与社交货币（MBTI化）**：
    *   提炼出极具社交网感的“趣味体质标签”（如：“朋克养生党”、“随时漏电的小电池”），内置一键生成分享海报功能，驱动微信生态内的自然裂变。
*   **用户反馈闭环（人类反馈）**：
    *   报告区嵌入【准】/【不准】反馈机制，错判数据自动回流，用于后续知识库的持续微调优化。

### 3. 康养方案生成智能体（Agent 3：The Planner）
**【模块定位】**：千人千面的动态生活建议引擎。
**【业务目标】**：提供低门槛、高执行率的健康管理日常指南。

*   **多维生活干预矩阵**：
    *   输出三个明确维度的建议：🍲 饮食建议（宜/忌食材）、🧘 起居运动（睡眠建议/拉伸动作）、🍵 居家理疗（日常按揉/茶饮配方）。
*   **逆向心理学设计（反向高亮）**：
    *   方案顶部强制呈现“红线警示”（今日三忌）。例如：❌ 绝不能喝冰饮、🚫 避免剧烈运动。利用“不让你做什么”的设计提升方案的专业感。
*   **微习惯拆解（Micro-habits）**：
    *   将宏大的养生计划拆解为“每日一项 3 分钟微任务”（如：睡前 3 分钟揉按足三里），极大降低用户的行动阻力。
*   **动态调整机制**：
    *   支持用户提出异议（如：“我对海鲜过敏”），Agent 毫秒级重构规避冲突后的新方案。

### 4. 康养智能营销服务智能体（Agent 4：The Salesperson）
**【模块定位】**：懂情商的商业转化中枢。
**【业务目标】**：在不破坏用户体验的前提下，实现课程、商品、线下服务的精准分发。

*   **全场景无缝植入（Contextual Commerce）**：
    *   监听 Agent 3 的方案，当出现“舒缓精神”的建议时，无缝衔接后端的商品池，在对话流中下发原生商品卡片（带图文和购买链接）。
*   **价值驱动的定制化推荐（极度关键）**：
    *   强制 LLM 在推荐商品前输出“逻辑推理链”。例如：“结合您的【气虚体质】及【秋季易燥】的环境，特地为您匹配了这套【黄芪党参调理包】”，用定制化逻辑打消用户防备心。
*   **动态权益分发引擎**：
    *   结合后台营销策略，Agent 以“专属福利”口吻下发限时折扣券（如：首测专享50元瑜伽体验券），缩短用户的决策链路。

## 二、 账号转化与数据资产留存策略

本架构采用 **“体验前置，价值后置（Hook Model）”** 的增长策略，极大地保护初次点击进来的流量：

1. **零阻力进入（Guest Mode）**：
   *   用户点开 H5 链接无需注册，直接与 Agent 1 开始互动。数据通过前端 `LocalStorage` 与匿名 `UUID` 进行双重绑定保存。
2. **核心价值卡点（Loginwall）**：
   *   在价值密度最高的节点（即准备输出深度体质评估报告时），系统柔性弹窗：“您的专属深度健康档案已生成，为保证数据永久留存，请一键微信安全登录查看”。
3. **数据资产归户（Data Merger）**：
   *   获取授权后，后台异步服务将临时 UUID 名下的所有测试数据与上下文无损合并至正式用户画像中，实现极高转化率。
