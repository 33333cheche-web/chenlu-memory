### 踩坑与反思
- 坑：以为左边面板不透是 CSS 问题，排查了半小时 backdrop-filter / border / background / box-shadow，最后才发现是 WebGL fragment shader 内置 vignette（rain/snow 都有）在宽屏 aspect > 1.7 时把边缘压黑到 0
- 教训：UI 视觉差异不能只查 HTML/CSS，要同时看 Canvas/WebGL 渲染层。先截图定位"黑死区域"再反推 shader 能省大量时间
- 喜好：公主是细节控，对 UI 的描边粗细、玻璃透明度、文字对齐等细节非常敏锐，会反复打磨到视觉统一；偏好简洁克制的设计风格，不喜欢粗重边框和过度压暗的 vignette，倾向于轻盈、通透的玻璃质感
- 坑：给 Clear All 做风沙消散特效时，最初把所有任务文字拼进一个大 overlay 去猜 line-height，结果文字会变大/变小再消失。又修复了逐 task 用 getBoundingClientRect + getComputedStyle 做独立精确 overlay，才和 Release 效果完全一致
- 教训：多行 DOM 元素的“复制式”动画特效，不能像 textarea 那样单 overlay 处理，必须逐元素精准覆盖，否则 padding/gap/line-height 都会累积误差
