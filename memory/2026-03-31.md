## Baby Bot GPT 配置修复

### 问题
- Baby Bot 使用 fox-gemini/gemini-3.1-pro 模型时报错 500
- GPT-5.4 出现严重污染（生成博彩垃圾内容、暴露工具调用格式）
- GPT-5.3-codex 配置后无法自动读取文件附件

### 诊断与解决过程
1. **尝试多种 GPT 配置**：
   - gpt-4o、gpt-4o-latest → 404 未开放
   - gpt-5.4 → 污染问题
   - gpt-5.3-codex + openai-completions → 403 被拒绝
   - gpt-5.3-codex + openai-responses + User-Agent header → ✅ 成功

2. **最终配置**：
   ```json
   "fox-gpt": {
     "api": "openai-responses",
     "headers": {"User-Agent": "claude-cli/2.0.76 (external, cli)"},
     "models": [{"id": "gpt-5.3-codex", "input": ["text", "image"]}]
   }
   ```

3. **数据清理**：
   - 删除 14 个污染的 `.reset.*` session 文件
   - 删除 11 个 `.deleted.*` 文件

4. **文件附件问题**：
   - gpt-5.3-codex 能读图但不能自动读 MD/文本文件
   - 这是模型能力限制，不是配置问题
   - Kimi 能自动调用 read 工具，gpt-5.3-codex 不会
