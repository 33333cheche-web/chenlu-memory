# 6 Bots 日报规范 v3.0

> 生效日期：2026-04-10
> 制定：Sunny
> 批准：公主

---

## 一、核心原则

**日报只写一件事：今天完成了什么。**  
不要写成工作日志、问题分析、或心情随笔。

---

## 二、文件位置（强制）

必须写入以下固定路径：

```
~/.openclaw/workspace-{bot}/memory/daily/YYYY-MM-DD.md
```

**⚠️ 禁止写入 `memory/YYYY-MM-DD.md` 根目录。**  
写入错误位置的日报，汇总时视为未提交。

---

## 三、格式规范（强制）

### 3.1 单条格式

每条日报独占一行，使用以下两种格式之一：

**格式 A（推荐）：**
```markdown
#P0 核心事项描述
#P1 工作事项描述
#P2 日常/临时事项描述
```

**格式 B（兼容）：**
```markdown
- [P0] 核心事项描述
- [P1] 工作事项描述
- [P2] 日常/临时事项描述
```

### 3.2 标签定义

| 标签 | 级别 | 保留时间 | 示例 |
|------|------|---------|------|
| **#P0** | 核心 | 永久 | `#P0 完成客户合同签署` |
| **#P1** | 工作 | 90天 | `#P1 优化推文排版` |
| **#P2** | 日常 | 30天 | `#P2 测试新配色方案` |

### 3.3 禁止内容

日报文件中**严禁**出现以下内容：

- ❌ 时间戳前缀（如 `## [09:00]`、`### 上午事件`）
- ❌ 大段说明文字或背景分析
- ❌ 碎碎念、反思、情绪表达
- ❌ emoji 前缀或装饰符号
- ❌ 多级标题结构
- ❌ 空行分隔的段落

**错误示例：**
```markdown
## 上午事件
### [10:00] Dashboard 问题
- 用户报告 dashboard 报错 Error 1033
- 原因是 Cloudflare tunnel 端口不对
- 建议用固定域名
```

**正确示例：**
```markdown
#P1 修复 dashboard Error 1033，重新配置 Cloudflare tunnel
#P1 创建 Cloudflare Named Tunnel 固定域名认证任务
```

---

## 四、碎碎念/备注放哪里？

如果必须记录非任务类内容，写入独立文件：

```
~/.openclaw/workspace-{bot}/memory/daily/YYYY-MM-DD_notes.md
```

该文件**不参与日报汇总**，仅作个人备忘。

---

## 五、汇总机制

| 时间 | 动作 |
|------|------|
| 22:05 | 各 Bot 生成并保存日报 |
| 22:22 | Sunny 自动汇总，生成 HTML 报表发送给公主 |

**未按规范提交的 Bot，汇总报表中将显示为空或标记"格式不合规"。**

---

## 六、检查清单

提交日报前，对文件内容做以下确认：

- [ ] 文件路径正确：`memory/daily/YYYY-MM-DD.md`
- [ ] 每条以 `#P0` / `#P1` / `#P2` 或 `- [P0]` / `- [P1]` / `- [P2]` 开头
- [ ] 没有时间戳、标题、emoji、大段说明
- [ ] 碎碎念已拆到 `_notes.md` 文件（如有）
